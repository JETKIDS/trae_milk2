## 前提
- 既存の計画（健診と動作確認）を実行に移す。月は当月（環境日付）を既定、コース指定は全コース（`all`）。必要に応じて後で調整。

## ステップ1: 依存導入/DB準備
- ルート→`server`→`client` の順に `npm install` を実行
- 必要なら `server/init.js` でサンプルデータ投入（既存DBが空の場合）

## ステップ2: 起動確認
- ルートで `npm run dev` を開始（サーバ9000／クライアント3000）
- `GET /api/health` で応答を確認し、`http://localhost:3000` のダッシュボード表示を確認

## ステップ3: 主要フロー健診
- 顧客詳細とカレンダー、臨時変更の表示/操作確認
- 月次請求: 確定→ステータス→解除→一括確定/解除のAPI応答を確認
- 入金: 個別/一括登録・一覧・メモ編集・取消・削除の応答とUIの連動確認
- 配達: 日別/期間別・商品合計・コース別集計APIの応答確認
- 分析: 月次売上・粗利・KPIの応答・主要指標の算出確認
- 口座振替: `GET /api/debits/generate?month=YYYY-MM&format=zengin_fixed` の120桁・SJIS出力確認

## ステップ4: 自動テスト
- クライアントE2E: `client` で `npm run test:e2e` を実行（`tests/e2e/collection_list_diff.spec.ts` 含む）
- サーバ統合: `server` で `npm test` を実行（KPI/固定長など）

## ステップ5: レポート
- 実行ログと成功/失敗の一覧を提示。改善候補（認証導入、マイグレーション明示化、監査ログ）を併記
