## 目的
- クライアント/サーバの起動と主要フロー（顧客詳細・月次請求・入金・配達・分析・口座振替）の健診
- 既存の自動テスト（Playwright/Jest）で回帰の土台を確認

## フェーズ1: 環境準備
- 依存導入: ルート→`server`→`client` の順で `npm install`
- DB初期化: `server/init.js` 実行で基本データ投入（必要時）

## フェーズ2: 起動確認
- ルートで `npm run dev` 起動（`3000`/`9000`）
- ヘルスチェック `GET /api/health` の応答確認

## フェーズ3: 主要API/画面健診
- 顧客詳細: 顧客取得と配達パターン・臨時変更の表示/編集
- 月次請求: 確定/解除・ステータス取得・一括確定/解除
- 入金: 個別/一括登録・一覧・メモ・取消・削除
- 配達: 日別/期間別・コース別商品合計
- 分析: 売上・粗利・商品/コース集計・月次KPI
- 口座振替: `format=zengin_fixed` と `zengin/csv` 出力の文字コード/行長確認

## フェーズ4: 自動テスト
- フロントE2E: `client` で `npm run test:e2e` 実行（スモークと主要UI）
- サーバ統合: `server` で `npm test` 実行（KPI/固定長など）

## フェーズ5: レポート
- 結果サマリ（成功/失敗、検証ログ）と改善候補（認証導入、スキーマ差分の明示マイグレーション、監査ログ）を提示
